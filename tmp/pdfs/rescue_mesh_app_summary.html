Rescue Mesh App

Rescue Mesh is an offline-first emergency communication app built around BLE SOS beacons, local storage, and delayed network sync. The repo also includes an optional Node.js command server, a Vue command dashboard, offline map support, and an on-device AI first-aid assistant.

Primary target Android field phone with BLE broadcast and scan support
Core stack Flutter + Riverpod + Drift/SQLite + BLE + optional Node/Vue services
Evidence base README, lib/, server/src/, dashboard/src/, pubspec.yaml

Who It's For

Primary persona: a person in a disaster or disconnected rescue scene carrying an Android phone, either as an SOS sender or as a passerby "data mule" relay node that helps move alerts toward connectivity.

Repo evidence points to two field roles: the distressed user broadcasting SOS and nearby users scanning, relaying, caching, and later uploading those records.

What It Does

  • Triggers SOS with a live location fix, saves it locally, and starts BLE distress broadcast.
  • Scans nearby BLE rescue payloads, decodes them, de-duplicates them, and stores them in SQLite.
  • Relays other devices' SOS payloads through a BLE relay queue for multi-hop propagation.
  • Auto-syncs pending local records to a backend endpoint when connectivity returns.
  • Shows nearby devices in radar UI and an offline tactical map backed by MBTiles.
  • Provides a Vue command screen with live alerts, map markers, blood-type counts, and trend stats.
  • Includes an offline AI first-aid chat and an ultra power-saving mode for field endurance.

How It Works

Mobile app: main.dart initializes PowerSavingManager, BleMeshService, BleScannerService, and NetworkSyncService, while Riverpod meshStateProvider feeds dashboard and radar UI.

SOS path: SosTriggerService gets location, writes to Drift/SQLite, starts BLE advertising, then tries immediate sync.
Scan path: BLE scan results are decoded, saved via saveIncomingSos(), added to the relay queue, and emitted to the UI after duplicate suppression.
Sync path: pending local records are POSTed to /api/sos/sync; Express/MongoDB merges or creates SosRecord documents and broadcasts Socket.IO updates.
Dashboard path: Vue fetches /api/sos/active, listens for new_sos_alert, and renders alert feed, Leaflet map, and ECharts metrics.

Offline map rendering uses MbtilesReader and falls back to online tiles when an MBTiles file is unavailable.

How To Run

  • Mobile app: flutter pub get then flutter run -d <device_id>.
  • Permissions: grant Bluetooth and location access on first Android launch.
  • Optional backend: cd server, npm install, create .env, then node src/index.js.
  • Optional dashboard: cd dashboard, npm install, then npm run dev.
  • When targeting your own server: run Flutter with --dart-define=API_BASE_URL=http://<LAN_IP>:3000.
